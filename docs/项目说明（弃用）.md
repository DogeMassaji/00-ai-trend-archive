# ~~GitHub AI Coding 趋势归档系统 - 完整教程~~

**项目代号**: AI-Trend-Archive  
**交付周期**: 7 天  
**难度**: ⭐⭐⭐ (适合有基础的大学生)  
**技术栈**: FastAPI + Vue 3 + SQLite  
**最终产物**: 可演示的全栈系统 + 完整工程化配置

---

## 🎯 项目定义与边界

### 核心价值主张
一个轻量级的**AI 编码趋势追踪与归档系统**,帮助开发者：
1. 定期抓取 GitHub 上 AI Coding 相关的热门项目
2. 分类归档并生成趋势报告
3. 提供搜索和订阅功能

### 明确边界（我们做什么）
✅ **做**：
- GitHub 仓库数据采集（通过 GitHub API）
- 项目分类管理（AI框架/IDE插件/代码生成/教程等）
- 趋势分析（Star增长、热度排行）
- 简单的标签系统
- 定时任务（每日更新数据）
- 用户收藏与笔记功能

❌ **不做**（控制范围）：
- 不做复杂的 AI 模型训练
- 不做实时数据流（只做定时批处理）
- 不做多用户权限管理（单用户或简单角色）
- 不做社交功能（不做评论、点赞、分享）
- 不做移动端适配（只做 PC Web）
- 不做国际化（只支持中文）

### 技术约束
- **数据库**：SQLite（方便部署，单文件）
- **后端**：FastAPI（快速开发，自动文档）
- **前端**：Vue 3 + Vite + TypeScript + Element Plus
- **定时任务**：APScheduler
- **数据源**：GitHub REST API（免费额度 5000次/小时）
- **图表**：ECharts

## 📦 核心功能（MVP 范围）

### 1. 项目采集模块
**功能描述**：
- 通过 GitHub API 搜索 AI Coding 相关项目
- 自动解析和存储项目信息
- 支持手动触发和定时自动采集

**核心流程**：
```
输入：搜索关键词（如 "AI coding assistant"）
    ↓
调用 GitHub Search API
    ↓
解析返回的仓库列表
    ↓
去重检查（避免重复收录）
    ↓
保存到数据库（新增或更新）
    ↓
输出：同步报告（新增X个，更新Y个）
```

**关键指标**：
- 仓库名称、描述、URL
- Star数、Fork数、Issue数
- 主要编程语言
- 创建时间、最后更新时间

### 2. 分类管理模块
**预设分类体系**：
1. **AI 编码助手**：Copilot、Cursor、Tabnine类工具
2. **代码生成工具**：文本转代码、低代码平台
3. **IDE 插件**：VS Code、JetBrains插件
4. **教程与资源**：学习资料、最佳实践
5. **开源模型**：代码生成模型、embedding模型
6. **测试与质量**：AI辅助测试、代码审查

**分类逻辑**：
- 基于关键词匹配（规则引擎）
- 支持手动调整
- 一个项目可属于多个分类

### 3. 趋势分析模块
**核心指标**：
- **日增量**：每日 Star 增长数
- **周/月排行榜**：时间段内增长最快的项目
- **活跃度评分**：综合 Star、Fork、最近更新时间计算

**展示形式**：
- 趋势折线图（最近30天）
- TOP 10 热门项目卡片
- 分类热度对比图

### 4. 搜索与筛选
**支持的筛选条件**：
- 关键词搜索（名称、描述）
- 分类筛选（多选）
- 编程语言筛选
- Star 数区间
- 更新时间范围

**排序方式**：
- 按 Star 数
- 按最近更新时间
- 按收录时间
- 按增长速度

### 5. 用户个人功能
**收藏系统**：
- 收藏感兴趣的项目
- 添加私人笔记（为什么收藏、使用心得）
- 查看收藏历史

**导出功能**：
- 导出为 Markdown（适合做周报、分享）
- 导出为 JSON（适合数据分析）

## 🗂️ 数据模型设计

### 核心表结构

#### 1. projects（项目表）
```sql
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    github_id INTEGER NOT NULL UNIQUE,           -- GitHub仓库ID
    name VARCHAR(255) NOT NULL,                  -- 仓库名
    full_name VARCHAR(255) NOT NULL,             -- 完整路径（如 github/copilot）
    description TEXT,                            -- 项目描述
    url VARCHAR(512) NOT NULL,                   -- GitHub链接
    homepage VARCHAR(512),                       -- 项目主页
    language VARCHAR(50),                        -- 主要语言
    stars INTEGER DEFAULT 0,                     -- Star数
    forks INTEGER DEFAULT 0,                     -- Fork数
    open_issues INTEGER DEFAULT 0,               -- 未关闭Issue数
    created_at DATETIME NOT NULL,                -- GitHub创建时间
    updated_at DATETIME NOT NULL,                -- GitHub最后更新
    first_seen DATETIME NOT NULL,                -- 首次收录时间
    last_synced DATETIME NOT NULL,               -- 最后同步时间
    is_archived BOOLEAN DEFAULT FALSE            -- 是否已归档
);

CREATE INDEX idx_projects_stars ON projects(stars DESC);
CREATE INDEX idx_projects_updated ON projects(updated_at DESC);
CREATE INDEX idx_projects_language ON projects(language);
```

#### 2. categories（分类表）
```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL UNIQUE,           -- 分类名
    slug VARCHAR(100) NOT NULL UNIQUE,           -- URL标识
    description TEXT,                            -- 分类说明
    icon VARCHAR(50),                            -- 图标类名
    sort_order INTEGER DEFAULT 0                 -- 排序权重
);

```

#### 3. project_categories（项目-分类关联表）
```sql
CREATE TABLE project_categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    category_id INTEGER NOT NULL,
    confidence REAL DEFAULT 1.0,                 -- 分类置信度（0-1）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE CASCADE,
    UNIQUE(project_id, category_id)
);

CREATE INDEX idx_project_categories_project ON project_categories(project_id);
CREATE INDEX idx_project_categories_category ON project_categories(category_id);
```

#### 4. project_snapshots（项目快照表）
```sql
CREATE TABLE project_snapshots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    stars INTEGER NOT NULL,
    forks INTEGER NOT NULL,
    open_issues INTEGER NOT NULL,
    snapshot_date DATE NOT NULL,                 -- 快照日期
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    UNIQUE(project_id, snapshot_date)
);

CREATE INDEX idx_snapshots_project_date ON project_snapshots(project_id, snapshot_date DESC);
```

#### 5. user_favorites（用户收藏表）
```sql
CREATE TABLE user_favorites (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    user_note TEXT,                              -- 用户笔记
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

CREATE INDEX idx_favorites_created ON user_favorites(created_at DESC);
```

#### 6. sync_logs（同步日志表）
```sql
CREATE TABLE sync_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sync_type VARCHAR(20) NOT NULL,              -- full/incremental
    projects_added INTEGER DEFAULT 0,
    projects_updated INTEGER DEFAULT 0,
    success BOOLEAN NOT NULL,
    error_message TEXT,
    started_at DATETIME NOT NULL,
    completed_at DATETIME,
    duration_seconds REAL
);

CREATE INDEX idx_sync_logs_started ON sync_logs(started_at DESC);
```

## 🔌 API 设计（后端接口）

### RESTful API 规范

**基础路径**：`/api/v1`  
**认证方式**：暂不需要（单用户模式）  
**返回格式**：统一 JSON  
**错误格式**：
```json
{
  "detail": "错误描述",
  "code": "ERROR_CODE"
}
```

sort: str = "stars"              # 排序字段
order: str = "desc"              # asc/desc

## 💡 常见问题与解决方案

### Q1: GitHub API 超限怎么办？
**问题**：未认证用户每小时只有 60 次请求。

**解决方案**：
1. 申请 Personal Access Token（免费，5000次/小时）
2. 实现指数退避重试
3. 缓存响应（1小时内不重复请求同一项目）
4. 监控 rate limit 剩余次数

**代码示例**：
```python
# utils/github_client.py
async def check_rate_limit(self):
    response = await self.client.get(
        "https://api.github.com/rate_limit",
        headers={"Authorization": f"token {self.token}"}
    )
    data = response.json()
    remaining = data["rate"]["remaining"]
    if remaining < 100:
        logger.warning(f"GitHub API rate limit low: {remaining}")
```

### Q2: 如何处理 GitHub 返回的数据缺失？
**问题**：某些字段可能为空（如 homepage、description）。

**解决方案**：
1. Pydantic 模型使用 `Optional`
2. 数据库字段允许 NULL
3. 前端显示时提供默认值

**代码示例**：
```python
# schemas/project.py
class ProjectResponse(BaseModel):
    name: str
    description: Optional[str] = None
    homepage: Optional[str] = None
```

### Q3: 定时任务如何调试？
**问题**：每天执行一次，等待时间太长。

**解决方案**：
1. 开发环境改为 5 分钟执行一次
2. 提供手动触发接口
3. 详细日志记录
4. 单独测试同步函数

**代码示例**：
```python
# tasks/scheduler.py
if settings.ENV == "development":
    scheduler.add_job(sync_task, 'interval', minutes=5)
else:
    scheduler.add_job(sync_task, 'cron', hour=2)
```

### Q4: 前端如何处理大量数据渲染？
**问题**：项目过多导致页面卡顿。

**解决方案**：
1. 使用分页（后端限制 page_size 最大 100）
2. 虚拟滚动（Element Plus `el-table` 支持）
3. 图片懒加载
4. 防抖/节流优化

### Q5: 如何保护 GitHub Token？
**问题**：Token 泄露会导致 API 额度被滥用。

**解决方案**：
1. 使用 `.env` 文件，加入 `.gitignore`
2. 后端不要在响应中返回 Token
3. 日志中脱敏
4. 定期轮换 Token
